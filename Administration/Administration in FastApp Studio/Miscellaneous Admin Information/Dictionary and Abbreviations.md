# Dictionary and Abbreviations

The following is a list of terms, their descriptions, and their abbreviations as they appear in the Profisee Platform and Support products.

| | | |
| --- | --- | --- |
| **Term** | **Description** | **Abbreviation** |
| Activity | An object configured in Visual Studio which perform a function according to its code and configuration. | - |
| Address Verification | A process that ensures address data input to Profisee Platform is valid. | AV |
| Amazon Web Services | A cloud-based deployment option provided by Amazon Inc. | AWS |
| Azure Kubernetes Services | A cloud-based deployment and storage option provided by Azure. This is the most common PaaS deployment option for Profisee customers. | AKS |
| Command Line Utility | An option that allows Profisee users to perform actions through the command line. | CLU |
| Data Quality Rule | Provides the ability to specify characteristics of what defines good data. | DQR |
| Domain-based Attribute | An attribute with values that are populated by records from another entity. | DBA |
| Eventing (Also referred to as "real-time event processing") | The Profisee feature area that allows administrators to configure event triggers, either from Profisee or from an external system. | - |
| Hierarchy | An arrangement of entity relationships into a tree structure that supports the ability to visualize, navigate, and process data in those entities in a top-down or bottom-up manner. | - |
| Lookup-Before-Create | A process that helps prevent accidental duplication of data by searching for possible matches in data before creating a new record. | LuBC |
| Many-to-Many Relationship | A relationship between two entities where one entity contains a parent instance for which there are several children in the other entity. | M2M |
| Master Data Management | The overall process of the improvement, consolidation, and accessibility of data in an organization. | MDM |
| Matching | The process of grouping multiple records that identify the same data. | - |
| Modeling | The process of creating a data model. | - |
| Multi-level Attribute | An attribute that can have more than one value associated with the entity. | MLA |
| Platform-as-a-Service | A platform deployment process performed entirely through a cloud-based service. | PaaS |
| Portal | The web-based interface for Profisee, primarily used by data stewards. | - |
| Presentation View | Reusable entity grid configurations for use with portal page controls in Profisee Portal. | PV |
| Software-as-a-Service | Software that exists primarily through cloud-based services. | SaaS |
| Staging | The process used to perform bulk loads of data to a model. | - |
| Survivorship | The process that creates golden records containing the most accurate data from the matching process. | - |
| Workflow | A long-running process composed of activities. | WF |
