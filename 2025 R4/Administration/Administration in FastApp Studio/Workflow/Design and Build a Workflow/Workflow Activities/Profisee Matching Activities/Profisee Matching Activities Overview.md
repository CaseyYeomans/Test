# Profisee Matching Activities Overview

The **Profisee Matching** activities that are available in the Visual Studio Toolbox provide the workflow designer the tools they need request various matching and look-up-before-create operations.

The table below lists the available activities and provides a summary of their functions and purpose. Refer to each activity in the table above for more specific usage and configuration details.

| | |
| --- | --- |
| **Activity** | **Description** |
| [LookUpRecord](https://support.profisee.com/wikis/profiseeplatform/look_up_member) | Looks up a [Record](http://support.profisee.com/wikis/profiseeplatform/member_data_type) from an entity based on fuzzy matching criteria leveraging the capabilities of a specific matching strategy. |
| [MatchRecord](https://support.profisee.com/wikis/profiseeplatform/match_member_activity) | Request that a [Record](http://support.profisee.com/wikis/profiseeplatform/member_data_type) be matched incrementally by being processed by the specified matching strategy. |
| [ProcessMatchingStrategy](https://support.profisee.com/wikis/profiseeplatform/process_matching_strategy) | Requests execution of a given matching strategy using the specified **ProcessAction** passed as an argument. |
| [SurviveMatchGroup](https://support.profisee.com/wikis/profiseeplatform/survive_match_group) | Requests execution of the survivorship rules for a specific match group. |
| [SurviveMatchGroups](https://support.profisee.com/wikis/profiseeplatform/survive_match_groups) | Requests execution of the survivorship rules for a collection of match groups. |
