# Address Verification

The table below describes the options and functions available for Address Verification in System Settings.

| Option | Function |
| --- | --- |
| Address Verification batch size | The number of address members to verify per batch. A number between 100 and 500 is recommended. |
| HTTP proxy URL | URL of the HTTP proxy server used to route address service requests for services with HTTP URLs, e.g. Bing and Loqate. |
| HTTPS proxy URL | URL of the HTTPS proxy server used to route address service requests for services with HTTPS URLs, e.g. Melissa Data and Google Maps. |
| Location verification maximum call rate | Location verification service provider maximum API calls allowed per minute. The default is 0 (unlimited). |
| Service Provider Keys | Used for verification service providers. You must reconnect after changing these values. |
| - Bing API | Location |
| - Google API | Location |
| - Loqate API | Mailing address |
| - Melissa Data API | Mailing address, phone, name, email verification |
